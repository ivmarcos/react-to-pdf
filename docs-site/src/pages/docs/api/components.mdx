      </button>

        ref={pdfRef}
        preview="children"
        page={{ format: 'a4', margin: Margin.MEDIUM }}

            This block is captured into the file when the user clicks

          </p>
        </div>
      </PDF>
    </>
  );
}`;

# Components API

The <span class="text-primary-600 dark:text-primary-400 font-bold">PDF</span>
component is a declarative alternative to the `usePDF` hook. You wrap the
content you want captured, attach a `ref`, and trigger generation
imperatively via the handle.

### Props

| Prop        | Type                                                        | Default            | Description                                                                   |
| ----------- | ----------------------------------------------------------- | ------------------ | ----------------------------------------------------------------------------- |
| `ref`       | `Ref<PDFHandle>`                                            | (none)             | Attaches the imperative handle (`save`, `open`, `print`, `update`, `getPdf`). |
| `preview`   | `false \| true \| "embed" \| "children"`                    | `false`            | How to preview the content while the PDF is built.                            |
| `page`      | `PageOptions`                                               | `{}`               | Page format / orientation / margin.                                           |
| `engine`    | `"canvas" \| "html"`                                        | `"canvas"`         | Which rendering engine to use (see [Rendering Engines](/react-to-pdf/docs/advanced/engines)). |
| `header`    | `({ page, pages }) => ReactNode \| FooterHeaderProps`       | (none)             | Renders on every page.                                                        |
| `footer`    | `({ page, pages }) => ReactNode \| FooterHeaderProps`       | (none)             | Renders on every page.                                                        |
| `canvas`    | `CanvasEngineOptions`                                       | `{}`               | Canvas-engine-specific options (resolution, align, size, hooks, …).           |
| `html`      | `HtmlEngineOptions`                                         | `{}`               | HTML-engine-specific options (autoPaging, fragmentScale, …).                  |
| `embedProps`| `HTMLEmbed` attributes                                      | `{ width: "100%", height: 400 }` | Forwarded to the `<embed>` when `preview="embed"`.                            |
| `loading`   | `ReactNode`                                                 | (none)             | Shown while the first build is in flight (with `preview="embed"`).            |
| `children`  | `ReactNode`                                                 | Required           | The content to capture into the PDF.                                          |

### Imperative handle: `PDFHandle`

```ts
interface PDFHandle {
  save(filename?: string): Promise<void>;
  open(): void;
  print(): void;
  update(): Promise<void>;      // re-run the build, e.g. after children change
  getPdf(): jsPDF | undefined;  // the raw jsPDF instance
  getDocument(): Document | undefined; // deprecated alias for getPdf()
}
```

### Preview modes

- `preview={false}` (default): the children are rendered offscreen. You see
  only the button; the PDF comes out when you call `save()` / `open()`.
- `preview="children"`: children are rendered inline. Use this when you
  want the user to see the content right on the page before downloading.
- `preview="embed"`: the generated PDF is embedded via `<embed>`. A preview
  of the *actual* PDF file, updated after each build.
- `preview={true}`: alias for `"embed"`.

## Multi-page documents with header and footer

The `header` and `footer` props accept a render function that receives
`{ page, pages }` on every page, so you can show dynamic page numbers. They
work identically on both engines.

The header/footer render function is mounted into an offscreen React root,
rasterised via html2canvas once per page, and stamped at the reserved top
and bottom margins, so any React component is supported, including custom
fonts, gradients, and SVG logos.

## Advanced: structured `<Body>`

If you need multiple independent body sections (e.g. sibling articles that
should each start on a new page), wrap them in the `<Body>` component:

```tsx
import { PDF, Body, Margin } from "react-to-pdf";

<PDF preview="children" page={{ margin: Margin.MEDIUM }}>
  <Body>{firstArticle}</Body>
  <Body startOnNewPage>{secondArticle}</Body>
  <Body startOnNewPage>{appendix}</Body>
</PDF>;
```

`<Body>` uses the same `PDFContext` the component relies on internally to
register each target element.

> The html engine currently supports a single body. Use the canvas engine
> when you need multiple `<Body>` children.

## When to reach for `<PDF>` vs `usePDF`

- Reach for `<PDF>` when you want the component to own the target element
  and you like declaring header/footer as props. It's the terser surface
  for simple documents and invoices.
- Reach for `usePDF` when you want finer-grained control, to share one ref
  across several buttons, or to build the PDF lazily from data that doesn't
  live in render output.
